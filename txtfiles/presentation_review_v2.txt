1- Problem Statement 
◻ Huge Staff Shortages: Most 911 centers are missing over 80% of the people 
they need. The few workers left are totally exhausted.
◻ Too Much Pressure: Because they’re overwhelmed by calls, it’s easy for 
operators to make mistakes or take longer to send help.
◻ Missing the Big Picture: Since they can only hear audio, it’s hard for 
dispatchers to judge how bad a situation really is based on a caller's tone.
◻ Too Much Manual Work: Typing everything in by hand wastes precious 
seconds that people in an emergency don't have.

2- Business Objectives 
Emotion-Aware Emergency Dispatch System DS-FYP    DHA Suffa University 
◻ Operational Efficiency: Handle 30% more calls with existing staffing levels by 
automating data entry and triage.
◻ Cost Reduction: Eliminate recurring Cloud API fees by pivoting to a one-time 
Local GPU investment.
◻ Performance Targets:
◻ Latency: Transcription < 2 seconds.
◻ Uptime: 99.9% (Fully offline-capable).
◻ Accuracy: >80% alignment with human emotion consensus.

3- Project Scope 
Scope Includes: • Real-time call transcription (Whisper) • Emotion detection using text & voice        (BERT + OpenSMILE) • Response generation using Fine-Tuned LLM • Dispatcher dashboard with map and live chat • Free open-source deployment (no paid APIs)Scope Excludes: • No real ambulance dispatch • No Multilingual support • No integration with real 911 systems  (due to government control, cost, API limitations)
